Brukerveiledning for Packet Tracer

Hva er Packet Tracer?

Packet Tracer er et simulert nettverksverktøy utviklet av Cisco som lar brukere lage, konfigurere og teste nettverksscenarioer i et virtuelt miljø. Det er mye brukt i utdanning og opplæring innen nettverksadministrasjon, spesielt for de som studerer Cisco-sertifiseringer som CCNA. Har du ikke nedlastet Cisco Packet Tracer? Klikk herher for nedlasting!

Hvordan fungerer Packet Tracer?

Packet Tracer simulerer Cisco-routers, switches, PC-er og andre nettverkselementer. Du kan dra og slippe enheter inn i et virtuelt arbeidsområde, koble dem sammen med virtuelle kabler, og konfigurere dem via kommandolinje- eller grafiske grensesnitt. Det gir en trygg og billig måte å lære nettverkskonfigurasjon og feilsøking på.

Hvordan lage en VLAN i Packet Tracer — enkel og tydelig

Følgende trinn er korte og klare: jeg viser hva du gjør, hvorfor det gjøres, og en kopi-vennlig kommando for CLI i Packet Tracer.

  1. Forberedelser
    Hva: Start Packet Tracer og legg inn en switch (f.eks. 2960), en ruter (valgfritt) og 1–4 PC-er. Hvorfor: Du trenger fysiske enheter i laben for å simulere VLAN.
  2. Opprett VLAN på switchen
    Hva: Definer et VLAN (f.eks. VLAN 10). Hvorfor: Segmenterer trafikk i eget broadcast-domene. 
    enable
configure terminal
vlan 10
 name VLAN10
exit
  3. Tildel access-porter
    Hva: Sett portene som access og koble PC-er til riktig VLAN. Hvorfor: Sikrer at endepunkter blir medlem av riktig VLAN. 
    interface FastEthernet0/1
 description PC0 - VLAN10
 switchport mode access
 switchport access vlan 10
 spanning-tree portfast
 exit
  4. Konfigurer uplink som trunk
    Hva: Sett uplink-porten til trunk for å transportere VLAN-tagget trafikk videre. Hvorfor: Nødvendig hvis ruter eller annen switch skal håndtere flere VLAN. 
    interface GigabitEthernet0/1
 description Uplink to Router/Switch
 switchport mode trunk
 switchport trunk allowed vlan 10
 exit
  5. Opprett default gateway (to alternativer)
    Alternativ A — Router-on-a-stick (ruter bruker subinterfaces): 
    interface GigabitEthernet0/0.10
 encapsulation dot1Q 10
 ip address 192.168.0.254 255.255.255.0

interface GigabitEthernet0/0
 no shutdown
    Alternativ B — SVI på Layer-3 switch: 
    ip routing
interface Vlan10
 ip address 192.168.0.254 255.255.255.0
 no shutdown
    Velg én gateway per VLAN (i eksemplene: 192.168.0.254 for VLAN10).
  6. Sett IP på PC-er
    Hva: Gi PC-er IP i VLAN-subnettet og sett default gateway. 
    På PC (Desktop → IP Configuration):
IP address: 192.168.0.1
Subnet mask: 255.255.255.0
Default gateway: 192.168.0.254
  7. Sikkerhet og dokumentasjon (anbefalt)
    Hva: Legg beskrivelser og aktiver enkel port-security. 
    interface FastEthernet0/1
 switchport mode access
 switchport access vlan 10
 switchport port-security
 switchport port-security maximum 1
 switchport port-security violation restrict
 switchport port-security mac-address sticky
 description PC0 - VLAN10
  8. Verifisering — sjekkliste
    Kjør disse kommandoene for å bekrefte: 
    På switch:
 show vlan brief
 show interfaces status
 show interfaces trunk

På ruter eller L3-switch:
 show ip interface brief

Fra en PC:
 ping 192.168.0.254   (ping default gateway)
 ping en annen PC i samme VLAN
  9. Vanlige problemer
    • PC kan ikke pinge gateway — sjekk at port er access på riktig VLAN og gateway-IP er i samme subnett.
    • Ping mellom VLAN feiler — sørg for at inter-VLAN routing er konfigurert (router-on-a-stick eller SVI).
    • Trunk frakoblet — sjekk trunk-sidekonfigurasjon og at trunk tillater VLANet.

Tips: For større laber, bruk DHCP i Packet Tracer for automatisk IP-tildeling og dokumenter port-descriptions for enkel feilsøking.
